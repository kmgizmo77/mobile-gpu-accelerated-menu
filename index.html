GPU Accelerated Mobile Header

Some Background

Mobile browsers do not have a consistent fixed positioning support. Older mobile browser versions like default browsers on iOS 4 and earlier, Android 2.3 and earlier, WP 7 and Blackberry 5 for example, either completely ignore the position: fixed attribute and render the element with position: static or even if the browser supports position: fixed the performance is jittery.

Also, one must consider the fact that the viewport space available on a mobile device is very limited. Thus, the decision of having a fixed position header (or a footer) should be taken only if it is essential for a better user experience. If not, then the headers should be replaced with statically positioned elements that will scroll normally with the rest of the content.

Alternate Solution

The header presented here is similar to the address bar that you see in modern day mobile phone browsers or even some apps. If the user is scrolling down in a page, he intends to read more content. In that case, the header slides up and entire screen is available for the content. If the user is scrolling up, in most of the cases he intends to get back to the top or to the menu. In that case, the header slides back in to the view, as if it were a fixed position element.
